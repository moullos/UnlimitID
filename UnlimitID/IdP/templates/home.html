To use the IdP you must have a user's endpoint running, and a RP. For testing purposes use this RP from here. It is ready to use and already registered. If you are running the User and the RP on the same machine make sure they do not share cookies.
Then:
1. Register a valid user
2. Use the user's endpoint to get a credential and extract an RP-specific "show" file. The name of the RP is 'dev' in this case
3. Initiate an authorization flow by visiting the RP
4. Upload the show file and get the access token
5. Revisiting the RP will automatically redeem the token for user info
